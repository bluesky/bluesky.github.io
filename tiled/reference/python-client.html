These are functions for constructing a client object.

Special constructors¶

These are typically used for development and debugging only.

Build Catalogs directly, running the app in this same process.

Advanced: Connect to a Catalog using a custom instance of httpx.Client or httpx.AsyncClient.

Client Catalog¶

The Catalog interface extends the collections.abc.Mapping (i.e. read-only dict) interface, so it supports these standard “magic methods”:

  • __getitem__ (lookup with [])

  • __iter__ (iteration, use in for-loops for example)

  • __len__ (has a length, can be passed to len)

as well as:

tiled.client.catalog.Catalog.get(k[,d])

tiled.client.catalog.Catalog.keys()

tiled.client.catalog.Catalog.items()

tiled.client.catalog.Catalog.values()

It adds the following attributes

tiled.client.catalog.Catalog.metadata

Metadata about this data source.

tiled.client.catalog.Catalog.sorting

The current sorting of this Catalog

tiled.client.catalog.Catalog.path

Sequence of entry names from the root Catalog to this entry

tiled.client.catalog.Catalog.uri

Direct link to this entry

And it adds these methods, which return a new Catalog instance.

tiled.client.catalog.Catalog.search(query)

Make a Catalog with a subset of this Catalog’s entries, filtered by query.

tiled.client.catalog.Catalog.sort(sorting)

Make a Catalog with the same entries but sorted according to sorting.

Finally, it adds attributes that provide efficient positional-based lookup, as in catalog.values_indexer[500:600].

tiled.client.catalog.Catalog.keys_indexer

tiled.client.catalog.Catalog.items_indexer

tiled.client.catalog.Catalog.values_indexer

Finally, it exposes these methods, which are used internally and may be useful in advanced applications.

tiled.client.catalog.Catalog.client_for_item(…)

Create an instance of the appropriate client class for an item.

tiled.client.catalog.Catalog.new_variation(*)

Create a copy of this Catalog, optionally varying some parameters.

tiled.client.catalog.Catalog.touch()

Access all the data in this Catalog.

Structure Clients¶

For each structure family (“array”, “dataframe”, etc.) there is a client object that understand how to request and decode chunks/partitions of data for this structure.

In fact, there can be more than one client for a given structure family. Tiled currently includes two clients for each structure family:

  • A client that reads the data into dask-backed objects (dask array, dask DataFrame, xarray objects backed by dask arrays)

  • A client that reads the data into in-memory structures (numpy array, pandas DataFrame, xarray objects backed by numpy arrays)

Cache¶

The module tiled.client.cache includes objects inspired by https://github.com/dask/cachey/

We opted for an independent implementation because reusing cachey would have required:

  • An invasive subclass that could be a bit fragile

  • And also composition in order to get the public API we want

  • Carrying around some complexity/features that we do not use here

The original cachey license (which, like Tiled’s, is 3-clause BSD) is included in the same source directory as the tiled.client.cache module.
